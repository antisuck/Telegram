# Видео-файлообменник для Telegram

Простой бот: админы загружают видео, бот выдаёт уникальную ссылку, по которой
любой пользователь получает видео — с обратным отсчётом и автоудалением через 30 секунд.

## Как это работает

1. Админ присылает боту видео.
1. Бот сохраняет `file_id` видео в SQLite-базе (`files.db`) и генерирует
   уникальную ссылку вида `https://t.me/имя_бота?start=AbCdEf1234`.
1. Любой человек переходит по ссылке → бот присылает видео с подписью-счётчиком
   (сколько секунд осталось до удаления и сколько раз файл скачали по этой ссылке).
1. Каждые 5 секунд бот обновляет счётчик в подписи, а через 30 секунд удаляет
   сообщение с видео из чата получателя.

## Установка

1. Установите Python 3.10+.
1. Установите зависимости:
   
   ```bash
   pip install -r requirements.txt
   ```
1. Создайте бота через [@BotFather](https://t.me/BotFather), получите токен.
1. Узнайте свой Telegram ID (например, через [@userinfobot](https://t.me/userinfobot)).
1. Скопируйте `.env.example` в `.env` и заполните:
   
   ```
   BOT_TOKEN=ваш_токен_от_BotFather
   ADMIN_IDS=ваш_id,id_другого_админа
   ```

## Запуск

```bash
python bot.py
```

База данных `files.db` создастся автоматически при первом запуске.

## Настройки

В начале `bot.py`:

- `DELETE_AFTER = 30` — через сколько секунд удалять видео у получателя.
- `EDIT_STEP = 5` — как часто (в секундах) обновлять счётчик в подписи.
  Не ставьте слишком маленькое значение (например, 1) — Telegram ограничивает
  частоту редактирования сообщений и может начать отбрасывать запросы.

## Важные замечания

- **`file_id` не вечен.** Если вы удалите видео из самого бота/чата, где оно было
  загружено впервые, `file_id` обычно продолжает работать, так как Telegram
  хранит файл на своих серверах. Но если бот заблокирован/удалён, ссылки перестанут
  работать.
- **Удаление сообщения ≠ удаление файла из базы.** Ссылка остаётся рабочей и после
  истечения 30 секунд — по ней можно снова получить видео (со своим новым
  30-секундным таймером). Если нужна одноразовая ссылка (после первого
  использования — недействительна), добавьте проверку `downloads >= 1` в
  `send_video_by_link` и отказывайте в повторном доступе.
- **Хранение в БД хранит только `file_id`, а не сам файл** — это нормально и
  эффективно, реальный файл остаётся на серверах Telegram.
- Для продакшена рекомендуется запускать бота через systemd/Docker и сделать
  резервную копию `files.db`.

## Возможные доработки

- Команда `/stats` для админов — список загруженных файлов и количество скачиваний.
- Одноразовые ссылки (см. выше).
- Поддержка не только видео, но и других типов файлов (документы, аудио и т.д.) —
  достаточно добавить обработчик `F.document`, `F.audio` и хранить тип файла в БД.
- Настраиваемое время удаления для каждой ссылки отдельно.
